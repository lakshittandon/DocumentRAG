Reliable RAG Document Intelligence Platform

Final Year Project Implementation Plan
Student: Lakshit Tandon
Technology Stack: FastAPI, React, LangChain, Qdrant, Whoosh, PostgreSQL, Docker

Goal: Build a reliable document question-answering platform that retrieves evidence from a trusted corpus, generates citation-grounded answers, verifies support at sentence level, and provides measurable evaluation outputs for academic demonstration and report writing.

1. Problem Statement

Large language models can produce fluent answers but may hallucinate when the required information is not present in the prompt context. This project addresses that problem by building a reliable Retrieval-Augmented Generation platform that ingests a document corpus, performs hybrid retrieval, reranks evidence, produces citation-aware answers, verifies support, and exposes the full workflow through both API and user interface layers.

2. Objectives

  • Create a reproducible ingestion pipeline for PDF and text documents.
  • Implement dense retrieval, BM25 retrieval, Reciprocal Rank Fusion, and reranking.
  • Generate grounded answers with explicit citations and refusal behavior for unsupported questions.
  • Add sentence-level verification to reduce hallucination risk.
  • Build an evaluation harness with retrieval and answer quality metrics.
  • Deliver a working React demo, FastAPI backend, Docker deployment, and final report artifacts.

4. Module Breakdown

4.1 Ingestion and Parsing

Support upload and parsing of PDF and TXT files, extract text page-wise where possible, flatten basic tables to text, attach metadata such as filename, checksum, page, section, and chunk identifiers, and reject scanned PDFs in v1 when no extractable text is found.

4.2 Retrieval Pipeline

Chunk documents into overlapping windows of 500-800 tokens with 100-150 token overlap, run dense retrieval and BM25 retrieval in parallel, merge ranked outputs with Reciprocal Rank Fusion, rerank the top candidate set, and pass the strongest evidence set to the generator.

4.3 Grounded Generation

Enforce answer generation only from retrieved context, return explicit citations at document and page or section level, and use the exact refusal text Not found in the provided documents. whenever evidence quality is insufficient.

4.4 Verification and Evaluation

Verify answer sentences against the evidence set, compute support scores, flag unsupported claims, and run benchmark experiments reporting Recall@k, MRR, nDCG, citation correctness, refusal accuracy, and hallucination rate.

4.5 UI and Demo Layer

Deliver a complete demo application with login, corpus view, upload and reindex flows, question answering interface, source inspection, evaluation dashboard, and audit-log views suitable for viva and project review demonstrations.

6. Dataset Strategy

Use a public and reproducible corpus for development and evaluation so the project remains demonstrable without NDA restrictions. The seeded demo corpus will include policy-style, technical, and evaluation-oriented text documents. Additional public PDFs can be added later for stronger results and screenshots.

7. Evaluation Strategy

  • Retrieval: Recall@k, MRR, nDCG
  • Answer quality: keyword correctness, citation correctness, refusal correctness
  • Safety: hallucination rate and unsupported-sentence ratio
  • Comparisons: dense-only, BM25-only, hybrid, hybrid + rerank, hybrid + rerank + verification

8. Risks and Mitigations

Risk Impact Mitigation
Weak retrieval quality on mixed queries Poor answer grounding Use hybrid retrieval, chunk tuning, metadata, and reranking
Hallucinations despite retrieval Loss of trust Use strict prompting, refusal threshold, and sentence verification
Scanned PDFs without extractable text Ingestion failure Reject in v1 and document OCR as future work
Solo-project delivery pressure Time overruns Finish a usable MVP first, then improve polish and metrics
